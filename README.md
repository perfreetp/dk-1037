# 规则引擎试算评估台

一个用于产品、运营和风控人员的Web应用，用于在规则正式生效前验证规则变更对业务的影响。

## 功能特性

### 1. 规则选择页
- 规则版本管理：展示所有可用规则版本，支持版本对比
- 规则配置：设置规则参数和阈值
- 规则预览：查看规则内容和执行条件
- 规则链路追踪

### 2. 样本准备页
- 样本数据导入：支持Excel、CSV格式批量导入
- 样本预览和编辑：在线查看和修改样本数据
- 测试集管理：保存常用测试集，便于重复使用
- 样本分组：按人群、渠道、时间段等维度分组

### 3. 试算任务页
- 创建试算任务：关联规则版本和样本数据
- 任务参数配置：设置试算环境和模拟参数
- 批量任务管理：同时发起多个试算任务
- 任务状态监控：实时跟踪任务执行进度

### 4. 结果对比页
- 命中明细展示：详细列出每条样本的命中结果
- 未命中原因分析：解释样本未命中规则的原因
- 规则链路追踪：展示规则执行的全链路过程
- 版本差异对比：对比新旧版本的执行结果差异
- 分组统计分析：按人群、渠道、时间段分组查看

### 5. 评估报告页
- 评估结论生成：自动生成规则影响评估结论
- 上线建议提交：提交规则上线建议和风险评估
- 完整记录查看：查看历史所有试算记录
- 报告导出：生成PDF或在线报告文档

## 技术栈

- **前端框架**: React 18 + TypeScript
- **构建工具**: Vite
- **UI框架**: Tailwind CSS
- **路由管理**: React Router v6
- **状态管理**: Zustand
- **HTTP客户端**: Axios
- **数据可视化**: Recharts
- **图表组件**: TanStack Table
- **文件处理**: SheetJS (xlsx)
- **后端框架**: Express.js
- **数据库**: SQLite

## 快速开始

### 安装依赖

```bash
npm install
```

### 启动开发服务器

```bash
npm run dev
```

这将同时启动前端开发服务器（端口3000）和后端服务器（端口4000）。

### 构建生产版本

```bash
npm run build
```

## 使用说明

### 1. 规则选择
1. 在规则选择页面查看所有可用规则
2. 点击规则查看详情，包括规则条件和执行动作
3. 选择要试算的规则版本

### 2. 样本准备
1. 切换到样本准备页面
2. 拖拽或点击上传Excel/CSV文件
3. 选择样本，点击"保存为测试集"方便后续使用
4. 支持按用户群体、渠道等维度筛选

### 3. 创建试算任务
1. 切换到试算任务页面
2. 点击"新建任务"
3. 选择规则版本和样本数据
4. 设置模拟参数（时间范围、执行环境）
5. 创建任务并执行

### 4. 查看结果
1. 任务执行完成后，点击"查看结果"
2. 在结果对比页面查看详细分析
3. 切换不同标签页查看：命中明细、未命中分析、规则链路、版本对比
4. 支持导出报告

### 5. 生成评估报告
1. 切换到评估报告页面
2. 点击"新建评估"
3. 填写评估结论和上线建议
4. 提交审批

## 开发指南

### 添加新的页面
1. 在 `src/pages/` 下创建新的页面文件夹
2. 创建 `index.tsx` 文件
3. 在 `App.tsx` 中添加路由

### 添加新的组件
1. 在 `src/components/` 下创建组件
2. 使用 Tailwind CSS 进行样式设计
3. 从 `lucide-react` 导入图标

### API服务
- 所有API调用通过 `src/services/api.ts` 统一管理
- 使用Mock数据演示功能，无需后端支持
- 如需连接真实后端，修改 `USE_MOCK` 变量
