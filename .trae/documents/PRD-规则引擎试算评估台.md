# 规则引擎试算评估台 - 产品需求文档

## 1. 产品概述

规则引擎试算评估台是一款面向产品、运营和风控人员的Web应用，用于在规则正式生效前验证规则变更对业务的影响。通过模拟真实场景和样本数据，帮助团队在规则上线前发现潜在问题，降低业务风险，确保规则平稳迭代。

**核心价值**：
- 降低规则上线风险，提前识别问题
- 提高规则验证效率，支持批量试算
- 保留完整试算记录，便于追溯审计
- 促进跨部门协作，统一评估标准

**目标用户**：
- 产品经理：验证新规则设计是否符合预期
- 运营人员：评估规则调整对业务指标的影响
- 风控人员：识别规则漏洞和异常情况，确保风控有效性

## 2. 功能模块

### 2.1 核心页面

| 页面名称 | 功能概述 |
|---------|---------|
| 规则选择页 | 管理和选择规则版本，配置规则参数 |
| 样本准备页 | 导入和准备测试样本数据，保存常用测试集 |

## 3. 核心业务流程

### 3.1 试算评估主流程

```
开始 → 选择规则版本 → 配置规则参数 → 选择样本数据
  ↓
创建试算任务 → 执行试算 → 等待结果
  ↓
查看结果明细 → 对比新旧版本 → 分析差异原因
  ↓
标记可疑样本 → 生成评估结论 → 提交上线建议
  ↓
记录归档 → 结束
```

### 3.2 批量试算流程

```
选择多个规则版本 → 导入样本名单 → 配置模拟参数
  ↓
保存为测试集（可选）→ 批量创建试算任务
  ↓
批量执行 → 汇总结果 → 生成综合报告
  ↓
评估通过 → 提交上线建议 | 评估不通过 → 调整规则参数
```

## 4. 数据模型

### 4.1 核心实体

| 实体名称 | 描述 | 主要字段 |
|---------|------|---------|
| RuleVersion | 规则版本 | 规则ID、版本号、状态、配置参数 |
| SampleData | 样本数据 | 样本ID、数据内容、导入时间、标签 |
| TestSet | 测试集 | 测试集ID、名称、样本列表、创建时间 |
| TrialTask | 试算任务 | 任务ID、规则版本ID、样本范围、执行状态 |
| TrialResult | 试算结果 | 结果ID、命中状态、规则链路、异常标记 |
| EvaluationReport | 评估报告 | 报告ID、结论、风险等级、建议、创建时间 |
| TrialRecord | 试算记录 | 记录ID、任务快照、完整链路、归档状态 |

### 4.2 数据关系

- RuleVersion 1:N TrialTask
- TestSet 1:N SampleData
- TrialTask 1:1 TrialResult
- TrialResult 1:N TrialRecord
- EvaluationReport 1:N TrialRecord

## 6. 非功能性需求

### 6.1 性能需求
- 单个试算任务响应时间 < 5秒（样本量 ≤ 1000）
- 批量试算支持至少100个任务并发
- 页面加载时间 < 3秒
- 支持至少10000条样本数据处理

### 6.2 可用性需求
- 系统可用性达到 99.9%
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 关键操作支持撤销和重做
- 提供操作引导和帮助文档

### 6.3 安全需求
- 用户身份认证和权限控制
- 数据传输加密（HTTPS）
- 敏感数据脱敏处理
- 操作日志完整记录

### 6.4 扩展性需求
- 支持规则类型插件扩展
- 支持自定义字段和维度
- 支持与外部系统集成
- 支持数据导出和迁移
